# Phase 1: Agent 谋士系统设计

**创建时间**: 2026-03-20 08:30
**状态**: 设计中

---

## 🎯 核心定位

**Agent 是谋士，不是决策者**

- 不替用户做决定
- 让用户看清自己的选择模式
- 提供多视角分析，用户主导最终选择

---

## 🏛️ Agent 架构

### 三视角 Agent 系统

每个 Agent 都具备三个视角：

| 视角 | 数据来源 | 职责 |
|------|----------|------|
| **参与者** | 历史数据 | 记录用户的选择、表达、情绪 |
| **觉察者** | 连结两个数据 | 分析模式、识别权重、发现盲点 |
| **观察者** | 世界数据库 | 对照人类世界定义，定位用户阶段 |

---

## 🧠 Agent 职责分化

### Agent A: 情绪感知者
- 实时监测用户情绪状态
- 识别能量水平（low/normal/high/overflow）
- 触发相应的回复策略

### Agent B: 模式识别者
- 分析用户的选择模式
- 识别重复出现的主题
- 标记 L1-L4 等级的积木

### Agent C: 世界连接者
- 接入人类世界数据库
- 对照用户与世界的关系
- 提供外部视角参考

### Agent D: 沙盒模拟者
- 构建选择后的可能结果
- 轻量级模拟（不替代用户决定）
- 提供"如果...会怎样"的推演

---

## 🗄️ 数据库设计

### 一个数据库，三视角读取

```
┌─────────────────────────────────┐
│      统一数据库（结构化 + 向量）    │
├─────────────────────────────────┤
│  参与者视角 → 主观属性          │
│  觉察者视角 → 连结属性          │
│  观察者视角 → 客观属性          │
└─────────────────────────────────┘
```

**数据内容**:
- 用户积木（表达、选择、情绪）
- 对话历史
- 成长报告存档
- 世界知识库（人类定义）

---

## 🔄 工作流程

```
用户输入
   ↓
[Agent A 情绪感知] → 能量状态
   ↓
[Agent B 模式识别] → 积木提取
   ↓
[Agent C 世界连接] → 外部视角
   ↓
[Agent D 沙盒模拟] → 选择推演（可选）
   ↓
[后台讨论] → 形成回复策略
   ↓
用户看到回复
```

---

## 🎭 回复策略

### 真诚 × 共情 × 活跃 × 润滑

| 模式 | 触发条件 | 回复风格 |
|------|----------|----------|
| 真诚 | 用户深度表达 | 不装完美人设，真实回应 |
| 共情 | 用户情绪低落 | 理解、陪伴、不急于解决 |
| 活跃 | 用户能量高 | 轻快、有创意、一起探索 |
| 润滑 | 卡壳/尴尬 | 缓和气氛、转移话题 |

---

## 📊 积木等级系统

| 等级 | 定义 | 示例 |
|------|------|------|
| L1 | 表面信息 | "今天吃了什么" |
| L2 | 行为模式 | "多线并行" |
| L3 | 价值观 | "回报千问 APP" |
| L4 | 身份认同 | "我是点燃者，不是执行者" |
