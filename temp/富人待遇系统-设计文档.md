# 富人待遇系统 - 核心设计

## 哲学基础

> 同一个信息，三种视角，三个数据库

| 数据库 | 主观/客观 | 本质 |
|--------|-----------|------|
| 人类世界 DB | 人=主观，其他=客观 | 社会共识的最大公约数 |
| 个人 DB | 全部=主观 | 个人滤镜下的世界 |
| Agent DB | 全部=无属性 | 纯数据，等待被赋予意义 |

**关键洞察**：其实都是主观的，只是"主观的最大公约数"被当成了客观。

## 积木设计

**积木** = 关于用户的任何可识别信息

```
积木 {
  id: uuid
  原始信息：string
  时间戳：datetime
  来源：behavior|choice|expression|external
  
  // 三视图属性
  人类世界视角：{
    主观性：float (0-1, 人对社会的主观程度)
    客观标签：[tags] (社会共识分类)
    权重：float (社会重要性)
  }
  
  个人视角：{
    主观性：float (0-1, 个人滤镜强度)
    情感色彩：[emotion_tags]
    权重：float (对个人重要性)
  }
  
  Agent 视角：{
    主观性：0 (无属性)
    数据模式：[pattern_tags]
    权重：float (预测价值)
  }
  
  // 演化追踪
  版本历史：[{timestamp, changes}]
  被引用次数：int
  最后更新时间：datetime
}
```

**积木等级** = 用户选择时的权重
- L1: 瞬时行为（权重低，易变）
- L2: 习惯性选择（权重中，有模式）
- L3: 核心价值观（权重高，稳定）
- L4: 身份认同（权重极高，难改变）

## 实现优先级

### Phase 1: 最小可行系统
- [ ] 单数据库原型（先实现三视图逻辑，物理上可先一个 DB）
- [ ] 积木数据结构
- [ ] 1-2 个 Agent 原型
- [ ] 基础沙盒模拟（规则引擎）

### Phase 2: 多 Agent 扩展
- [ ] Agent 职责分化
- [ ] Agent 间对话/辩论机制
- [ ] 三数据库分离

### Phase 3: 成长系统
- [ ] 对比报告生成
- [ ] 可视化时间线
- [ ] 分享功能

### Phase 4: 动态适配
- [ ] 用户状态检测
- [ ] 场景自动切换
- [ ] Agent 策略进化

## 关键技术决策

1. **数据库**：PostgreSQL + pgvector（结构化 + 向量一体）
2. **Agent 框架**：基于 sessions_spawn 的 sub-agent 模式
3. **沙盒引擎**：规则引擎 + 轻量 LLM 推演
4. **前端**：待定（先 CLI/Web 原型）
