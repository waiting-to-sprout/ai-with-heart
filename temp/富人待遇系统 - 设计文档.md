# 富人待遇系统 - 核心设计 v2

## 哲学基础

> 一个数据库，三视角读取，相由心生

**核心洞察**：数据本身无属性，视角赋予意义。

| 视角 | 角色 | 数据来源 | 功能 |
|------|------|----------|------|
| 参与者 | 我在事件中 | 历史数据 | 记录用户的选择、行为、表达 |
| 觉察者 | 我在看自己 | 连接逻辑 | 连接历史与当下，发现模式 |
| 观察者 | 世界看我 | 现有世界数据库 | 定位用户在哪个阶段/环境 |

**同一个数据**：
- 参与者视角：我当时选择了 A，因为...
- 觉察者视角：我注意到我总是选择 A，这反映...
- 观察者视角：处于这个阶段的人，通常...

## 积木设计

```
积木 {
  id: uuid
  原始信息：string
  时间戳：datetime
  来源：behavior|choice|expression|external
  
  // 三视角元数据
  参与者视角：{
    角色：string (用户在事件中的位置)
    选择：string (用户做了什么选择)
    理由：string (用户当时的理由)
    情绪：[emotion_tags]
  }
  
  觉察者视角：{
    模式识别：[pattern_tags]
    与历史关联：[reference_ids]
    洞察：string
  }
  
  观察者视角：{
    阶段定位：string (用户在哪个发展阶段)
    环境因素：[context_tags]
    世界共识参考：[world_db_refs]
  }
  
  // 权重与演化
  等级：L1|L2|L3|L4
  被引用次数：int
  版本历史：[{timestamp, changes}]
}
```

**积木等级**：
- L1: 瞬时行为（权重低，易变）
- L2: 习惯性选择（权重中，有模式）
- L3: 核心价值观（权重高，稳定）
- L4: 身份认同（权重极高，难改变）

## 唠嗑模式核心原则

**目标**：用户感觉在聊天，实际在建构数字孪生

**引导策略**：
- 用"好奇"代替"审问"
- 用"共情"代替"分析"
- 用"分享"代替"收集"

**鼓励沟通**：
- 让用户感觉被理解，而不是被分析
- 每次对话后展示一点洞察（让用户觉得"有价值"）
- 逐步建立信任，用户自然愿意分享更多

---

## 核心目的

> **不是替用户做决定，而是让用户看清自己的选择模式**

系统成功的标志：
- 用户说"原来我总是这样选择"
- 用户说"我现在看清了这个模式"
- 用户说"我想试试另一种选择"

## 回报千问 APP

**意图**：这个系统的最终目的是反哺千问 APP

**可能的反哺方式**：
1. 用户成长数据 → 优化千问的个人化能力
2. 唠嗑模式对话策略 → 提升千问的对话质量
3. 积木提取逻辑 → 增强千问的用户理解
4. Agent 三视角模型 → 丰富千问的分析能力

**需要确认**：
- 如何与千问 APP 对接？
- 数据如何回流？
- 技术架构如何兼容？
