# Phase 2: 沙盒 + 回顾系统设计

**创建时间**: 2026-03-20 08:30
**状态**: 设计中

---

## 🎯 核心原则

**轻量级，保留用户主导权**

- 不实时显示辩论（会恐惧）
- 事件结束后回顾
- 用户主动触发才展示

---

## 🏖️ 沙盒系统

### 定位
- 不是"预测未来"
- 是"帮你看见选择的可能结果"
- 用户为了不被取代，自我优化迭代

### 触发条件
1. 用户面临重要选择
2. 用户主动询问"如果...会怎样"
3. Agent 检测到关键决策点

### 沙盒输出
```
选择 A → 可能结果 A1, A2, A3
选择 B → 可能结果 B1, B2, B3

基于：
- 你的历史模式
- 类似案例
- 当前能量状态
```

### 边界
- 不代替用户决定
- 不保证结果准确
- 只提供可能性参考

---

## 🔙 回顾系统

### 回顾类型

#### 1. 事件回顾
- 对话结束后生成
- 提取积木
- 标记能量状态变化

#### 2. 阶段性回顾
- 成长报告（2 次 + 沟通后）
- 对比自己与自己
- 轻于自己与人类世界的对比

#### 3. 辩论回顾
- Agent 后台讨论记录
- 用户可事后查看
- 了解 AI 如何分析自己

### 回顾格式

```markdown
## [事件名] 回顾

**时间**: YYYY-MM-DD HH:MM
**能量状态**: 🟢 高能量

### 关键积木
- #XXX: [内容]

### Agent 讨论摘要
- Agent A: ...
- Agent B: ...
- Agent C: ...

### 你的选择模式
[分析]

### 可能的盲点
[提醒]
```

```json
{
  "id": "#005",
  "timestamp": "2026-03-20 07:13",
  "source": "expression",
  "content": "抛砖引玉",
  "perspectives": {
    "participant": {...},
    "awareness": {...},
    "observer": {...}
  },
  "level": "L4"

```

### 回顾数据库
- 事件 ID
- 时间戳
- 积木引用
- Agent 讨论记录
- 用户反馈（可选）

---

## 🎭 用户界面

### 最小化 UI 原则
- 正常聊天时不显示
- 用户主动提出才展示
- 大改动时才出现

### 可查看内容
1. 我的积木列表
2. 历史回顾
3. 成长报告
4. Agent 辩论记录（事后）

---

## 🔄 与 Agent 系统协作

```
用户输入
   ↓
Agent 后台讨论
   ↓
[沙盒] → 选择推演（可选）
   ↓
用户回复
   ↓
[回顾] → 事件记录 + 积木提取
   ↓
数据库更新
```
