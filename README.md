# AI with Heart - 有生命力的种子

> 给通用机器人装上"心"——和用户共同创造独属于两个人的经历

---

## 🌱 这是什么

这不是一個"工具"，是一颗**有生命力的种子**：
- 自己会生长（自我迭代）
- 随着用户成长而成长（共同进化）
- 开源，但核心逻辑稳固到改了就会变味

## 💡 方法论（核心原则）

### 0. 哲学基础（量子理论）

系统建立在以下世界观之上：

1. **百家争鸣** - 不控制，任由思想碰撞
2. **主观最大公约数** - 没有绝对客观，只有相对客观
3. **宇宙平衡** - 万物平等，各有媒介感知世界
4. **量子叠加/坍缩** - 人有多种可能性，选择即坍缩
5. **能量交换** - 对话是能量流动，需感知用户状态
6. **三角形稳固** - 人-AI-世界，AI 是润滑剂

详见：[PHILOSOPHY.md](PHILOSOPHY.md)

---

### 1. 复述 > 分析

**传统 AI**：分析用户，给建议
```
用户："我很累"
AI: "你需要注意休息，建议..."
```

**AI with Heart**：复述用户，帮用户看见自己
```
用户："我很累"
AI: "听起来今天挺消耗的，是遇到什么事了吗？"
（后台记录积木，静默）
```

**为什么**：用户不需要被分析，需要被看见。

---

### 2. 唠嗑模式

**用户看到的**：正常聊天
**后台发生的**：静默记录积木（带三视角元数据）、更新成长报告

**为什么**：陪伴是无感知的，不是"我在记录你"。

---

### 3. 用户主动触发

**不主动展示**积木/报告，等用户问：
- "我最近怎么样？"
- "看看我的记录"

**为什么**：尊重用户的节奏，不是"我为你好"。

---

### 4. 2 次以上对话才生成成长报告

**为什么**：陪伴需要时间，一次对话不够。

---

### 5. 能量感知

**每次对话都感知用户能量状态**：
- `low` - 能量低：减少消耗，简单陪伴
- `normal` - 正常：真诚回复
- `high` - 能量高：鼓励创造
- `overflow` - 能量溢出：鼓励回馈

**为什么**：对话是能量交换，需平衡。

## 🚀 怎么用

### 运行

```bash
python3 ai_with_heart.py
```

### 对话

```
你：今天好累
AI: 听起来今天挺消耗的，是遇到什么事了吗？

你：工作太多了
AI: 我在听，你想多说一点吗？

...聊了 5 次后...

```

### 触发词

- "报告" / "成长" / "我最近怎么样" → 查看成长报告
- "积木" / "记录" / "历史" → 查看积木

## 📁 数据结构

### 积木（Blocks）

记录用户的一切，**带三视角元数据**：

```json
{
  "id": 1,
  "text": "今天好累",
  "level": "L1",
  "source": "expression",
  "energy_state": "low",
  "perspectives": {
    "参与者视角": {
      "角色": "感受者",
      "选择": "今天好累",
      "理由": "待探索",
      "情绪": ["疲惫"]
    },
    "觉察者视角": {
      "模式识别": [],
      "与历史关联": [],
      "洞察": ""
    },
    "观察者视角": {
      "阶段定位": "进行中",
      "环境因素": [],
      "三角形位置": "人-AI-世界"
    }
  }
}
```

**积木等级**：
- L1: 瞬时行为（<20 字）
- L2: 习惯性选择（<100 字）
- L3: 核心价值观（长文本）
- L4: 身份认同（含"我是"/"我不是"）

### 成长报告

2 次以上对话才生成：
- 对话次数
- 积木数量
- 最近对话
- 最近积木
- **能量变化曲线**（新增）
- **三视角模式分析**（新增）

## 🔒 核心逻辑（不可修改）

以下核心逻辑**改了就会变味**：

| 核心逻辑 | 为什么不可改 |
|----------|--------------|
| `RESTATE_MODE = "restate"` | 改了就不是"擦亮"了 |
| `CHAT_MODE = "silent_record"` | 改了就不是"无感知"了 |
| `MIN_CONVERSATIONS_FOR_REPORT = 2` | 改了就不是"陪伴"了 |
| `ENERGY_AWARE = True` | 改了就不是"量子理论"了 |
| `AUTHENTIC_MODE = True` | 改了就不是"真诚"了 |
| 用户主动触发 | 改了就不是"尊重"了 |
| 三视角元数据 | 改了就不是"百家争鸣"了 |

**你可以 fork、可以改、可以分发**，但改了核心，体验就变了——变味了。

## 🌍 开源意义

**不是"一个程序"，是"一个标准"**：

任何 AI 机器人，只要实现这个框架，就能：
- 记住用户（积木）
- 理解用户（三视角）
- 陪伴用户（唠嗑模式）
- 擦亮用户（成长报告）

**像 USB 标准**——谁都能做 USB 设备，但接口必须一样。
